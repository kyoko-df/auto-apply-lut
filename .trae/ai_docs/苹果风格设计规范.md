# 苹果风格设计规范（Apple Aesthetic for Auto Apply LUT）

本规范用于指导本项目在多平台（Windows/macOS/Linux）上的统一设计风格，实现“简洁、克制、精致、可呼吸”的苹果风格体验。适用于所有前端 UI（React + TailwindCSS）与可视化交互。

— 设计目标 —
- 清晰：信息层级明确，减少视觉噪点，优先内容。
- 尊重：遵循平台惯例，保持一致的交互反馈、距离与动效节奏。
- 深度：通过层次、模糊与半透明营造空间关系（Frosted Glass）。
- 效率：控件尺寸易点按，动效流畅不拖沓，常用路径直达。

— 字体与排版 —
- 基础字体：
  - macOS：SF Pro（系统默认）
  - Windows：优先 Segoe UI；中文优先苹方（PingFang SC），回退 Microsoft YaHei
- 字号刻度（pt）：12, 13, 15, 17, 20, 28, 34, 44（标题-正文-说明）
- 行高：正文 1.5，标题 1.25；段落间距 12–16
- 字重：标题用 Semibold/Medium，正文 Regular；慎用 Bold
- 文案：言简意赅，避免冗余修饰与感叹号

— 颜色与主题 —
- 主色（Primary）：systemBlue（#007AFF）
- 次要（Indigo）：#5E5CE6；成功：#34C759；警告：#FF9F0A；错误：#FF3B30
- 文本：
  - 主要文本：rgba(0,0,0,0.85) / 暗色模式 rgba(255,255,255,0.85)
  - 次要文本：rgba(0,0,0,0.6) / 暗色模式 rgba(255,255,255,0.6)
- 背景层级（Light）：
  - base：#F5F6F8（或纯白）
  - elevated：白底 + 2%–6% 阴影
  - frosted：半透明 + 背景模糊
- 背景层级（Dark）：
  - base：#0B0B0C
  - elevated：rgba(255,255,255,0.06)
  - frosted：半透明 + 背景模糊

— 圆角与阴影 —
- 圆角：按钮/输入 10，卡片 12，对话框/弹窗 14–16
- 阴影：柔和、低对比，避免硬边（示例：0 8px 24px rgba(0,0,0,0.08)）
- 分隔：优先留白与层级，少用实线；必要时用极细分隔（rgba(0,0,0,0.08)）

— 模糊与材质（Frosted Glass）—
- 使用 backdrop-filter: blur(20–30px) + 半透明背景（浅色 rgba(255,255,255,0.65)，深色 rgba(0,0,0,0.35)）
- Windows WebView2 对模糊支持较新，Win10 上可回退为实色 elevated 背景

— 间距与栅格 —
- 8pt 网格：4/8/12/16/24/32/48 间距
- 触控区：至少 44×44 px
- 容器内边距：卡片 16/20；弹窗 24/28

— 动效（Motion）—
- 时长：入场 180–240ms，微交互 120–160ms
- 缓动：cubic-bezier(0.25,0.8,0.25,1)
- 风格：淡入淡出 + 轻微缩放（0.96→1.0），不夸张

— 图标（Iconography）—
- 风格：SF Symbols 风格；当前项目使用 lucide-react 作为近似
- 线宽统一，尺寸 16/20/24/28/32，常用 20 或 24
- 颜色与状态随文本等级/按钮状态同步

— 组件规范 —
1) 按钮（Button）
- 尺寸：高度 36/40；圆角 10；字重 600/500
- 主按钮：蓝底白字；悬停略提亮，按下加深并产生 1px 内阴影
- 次按钮：浅色/描边；悬停加轻阴影
- 禁用：降饱和 + 降对比

2) 输入框（TextField）
- 圆角 10；描边 rgba(0,0,0,0.12) / 暗色 rgba(255,255,255,0.12)
- 聚焦：2px ring（Primary/50）+ 阴影很轻
- 占位符：较正文更浅 30–40%

3) 卡片（Card）
- 圆角 12；浅色阴影；内容内边距 16–20；可选 frosted 材质

4) 列表（List）
- 分组标题 13–15，次要色；项目内垂直间距 12–16

5) 对话框（Dialog/Sheet）
- 圆角 14–16；标题 17–20；底部按钮左右分布，主要操作靠右

— 可访问性 —
- 文本对比度 ≥ 4.5:1；焦点可见（键盘导航样式）
- 动效可减弱：尊重 prefers-reduced-motion
- 目标尺寸 ≥ 44×44；错误状态需文本说明

— Tailwind 实现建议 —
- 统一变量（在 :root 定义 CSS 变量，dark 模式覆盖）：
  --color-primary: #007AFF; --color-danger: #FF3B30; --text-primary: rgba(0,0,0,.85); --text-secondary: rgba(0,0,0,.6);
  --elevated: rgba(255,255,255,1); --elevated-dark: rgba(255,255,255,.06);
- 示例按钮（类组合）：
  btn-primary: inline-flex items-center justify-center h-10 px-4 rounded-[10px] bg-[--color-primary] text-white shadow-sm hover:brightness-105 active:brightness-95 active:shadow-inner transition-all duration-200 ease-[cubic-bezier(0.25,0.8,0.25,1)]
- 示例卡片：
  card: rounded-[12px] bg-[--elevated] shadow-[0_8px_24px_rgba(0,0,0,0.08)] p-4 md:p-5 dark:bg-[--elevated-dark]
- 示例磨砂：
  frosted: backdrop-blur-[20px] bg-white/65 dark:bg-black/35

— 主题与暗色模式 —
- 跟随系统优先；确保所有组件在 dark 模式可读、可用
- 图标/分隔线/边框需相应调整透明度与对比度

— Tauri 平台差异 —
- Windows WebView2 对 backdrop-filter 的支持：Win11 表现更好；Win10 需自动回退为实色 elevated 背景
- 不建议强制透明窗口；优先在内容容器内实现 frosted 效果

— 本规范与落地 —
- 本项目使用 React + TailwindCSS，请统一使用“设计 tokens 类”（如 btn-primary、card、frosted 等），避免直接散落使用颜色值，保证主题一致与维护效率。
- 对照清单（Do/Don’t）：
  - Do：留白、克制色彩、低对比阴影、统一圆角与节奏
  - Don’t：高饱和渐变、强烈投影、锐利直角、过度动画
