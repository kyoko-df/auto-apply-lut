## 核心模块说明

### 1. Rust后端模块 (src-tauri/src/)

#### commands/ - Tauri命令模块
负责前后端通信的命令定义，每个文件对应一个功能域：
- `file_manager.rs`: 文件选择、验证、信息获取、批量选择
- `lut_manager.rs`: LUT文件管理、解析、验证、收藏管理
- `processor.rs`: 批量处理控制、进度监控、暂停恢复
- `system.rs`: 系统信息、配置管理、性能监控
- `gpu_manager.rs`: GPU设备检测、管理、加速控制
- `batch_manager.rs`: 批量任务管理、优先级控制、重试机制

#### core/ - 核心业务逻辑
包含应用的核心功能实现：
- `video/`: 视频文件处理、批量选择、文件夹浏览相关逻辑
- `lut/`: LUT文件处理、库管理、收藏功能相关逻辑
- `ffmpeg/`: FFmpeg集成、GPU加速、命令优化
- `gpu/`: GPU设备管理、性能监控、加速控制
- `system/`: 系统资源监控、性能分析、推荐设置
- `task/`: 任务队列、调度管理、批量处理、暂停恢复
- `database/`: 数据库管理、数据访问对象、迁移脚本

#### types/ - 类型定义
统一的数据结构定义，确保类型安全：
- `video.rs`: 视频文件信息、格式、状态等类型
- `lut.rs`: LUT文件格式、参数、收藏等类型
- `task.rs`: 任务状态、进度、优先级等类型
- `gpu.rs`: GPU设备信息、能力、状态等类型
- `batch.rs`: 批量处理、并发控制等类型
- `system.rs`: 系统信息、性能监控等类型
- `error.rs`: 错误类型和错误处理

### 2. 前端模块 (src/)

#### components/ - React组件
按功能域组织的组件结构：
- `common/`: 可复用的通用组件（按钮、模态框、进度条等）
- `layout/`: 页面布局组件（头部、侧边栏、状态栏、系统监控）
- `file/`: 文件管理相关组件（列表、选择器、批量选择、文件夹浏览）
- `lut/`: LUT管理相关组件（选择器、库管理、收藏、搜索）
- `processing/`: 处理流程相关组件（面板、队列、进度监控、批量控制）
- `gpu/`: GPU相关组件（选择器、监控、设置、性能）
- `system/`: 系统监控组件（资源监控、性能图表、推荐设置）
- `settings/`: 设置界面组件（通用、输出、性能、GPU、并发、通知）

#### store/ - 状态管理
使用Zustand进行状态管理，按功能域分离：
- `fileStore.ts`: 文件列表、选择状态、批量操作
- `lutStore.ts`: LUT库、当前选择、收藏管理
- `processingStore.ts`: 处理队列、进度状态、批量控制
- `gpuStore.ts`: GPU设备状态、加速设置
- `batchStore.ts`: 批量处理状态、并发控制
- `systemStore.ts`: 系统监控数据、性能指标
- `taskStore.ts`: 任务管理、优先级、重试状态
- `settingsStore.ts`: 用户设置、偏好配置
- `notificationStore.ts`: 通知管理、消息状态

#### services/ - 服务层
封装与后端的通信逻辑：
- `tauri.ts`: Tauri API的统一封装
- `fileService.ts`: 文件管理、批量选择服务
- `lutService.ts`: LUT管理、收藏功能服务
- `processingService.ts`: 处理控制、进度监控服务
- `gpuService.ts`: GPU管理、加速控制服务
- `batchService.ts`: 批量处理、并发控制服务
- `systemService.ts`: 系统监控、性能分析服务
- `taskService.ts`: 任务管理、优先级控制服务
- `notificationService.ts`: 通知管理服务

## 开发工作流

### 1. 开发环境启动
```bash
# 安装依赖
npm install

# 初始化数据库
cd src-tauri && cargo run --bin init-db

# 启动开发服务器
npm run tauri:dev
```

### 2. 构建流程
```bash
# 开发构建
npm run build

# 生产构建（包含GPU加速支持）
npm run tauri:build -- --features vulkan

# 轻量版构建（不包含GPU加速）
npm run tauri:build
```

### 3. 测试流程
```bash
# 前端单元测试
npm run test

# 后端单元测试
cd src-tauri && cargo test

# GPU功能测试
cd src-tauri && cargo test --features vulkan gpu_tests

# 批量处理测试
cd src-tauri && cargo test batch_tests

# 集成测试
npm run test:e2e

# 性能测试
npm run test:performance
```

### 4. 数据库管理
```bash
# 创建新的数据库迁移
cd src-tauri && cargo run --bin create-migration <migration_name>

# 执行数据库迁移
cd src-tauri && cargo run --bin migrate

# 重置数据库
cd src-tauri && cargo run --bin reset-db
```

## 代码规范

### Rust代码规范
- 使用 `cargo fmt` 进行代码格式化
- 使用 `cargo clippy` 进行代码检查
- 遵循Rust官方命名约定
- 所有公共API必须有文档注释
- 异步函数优先使用 `async/await`
- 错误处理使用 `Result<T, E>` 类型
- GPU相关代码必须有fallback机制
- 数据库操作必须使用事务

### TypeScript代码规范
- 使用ESLint + Prettier进行代码检查和格式化
- 遵循React Hooks规范
- 组件必须有TypeScript类型定义
- 使用函数式组件和Hooks
- 状态管理使用Zustand
- 异步操作使用React Query或SWR
- GPU和系统监控组件必须处理数据缺失情况

### 数据库规范
- 所有表必须有主键
- 外键约束必须明确定义
- 索引必须根据查询模式优化
- 迁移脚本必须可回滚
- 敏感数据不得明文存储

### 性能规范
- 批量操作必须支持取消和暂停
- GPU检测失败必须graceful fallback
- 大文件处理必须显示进度
- 内存使用必须有上限控制
- 并发任务数必须可配置

### 提交规范
- 使用Conventional Commits格式
- 每个提交只包含一个功能或修复
- 提交信息必须清晰描述变更内容
- GPU相关变更必须标注兼容性
- 数据库变更必须包含迁移脚本
